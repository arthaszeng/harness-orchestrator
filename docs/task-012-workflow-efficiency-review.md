## 2) Plan/Ship 效率评审（重点结论）

结论：当前流程在治理可靠性上表现好，但效率仅达到**中上水平（约 7/10）**，尚未达到“足够高效”。

### 高优先级瓶颈（至少 3 项）

1. **判定标准不统一（P0）**
  - 影响类型：决策效率
  - 现象：`workflow 是否足够高效` 缺乏统一阈值，容易出现“结论不一致”
  - 建议：建立最小判定指标（e2e 耗时、人工介入次数、一次通过率）
2. **流程固定成本偏高（P1）**
  - 影响类型：交付时长
  - 现象：`ship` 中覆盖率审计、5 角色评审、多层 gate 对小改动成本较高
  - 建议：引入风险分层（低风险路径保留硬门禁但裁剪高成本分析）
3. **模板能力与本地命令可用性存在错位风险（P1）**
  - 影响类型：可执行性
  - 现象：模板可能先于本地命令入口，导致“文档可做、命令不可用”
  - 建议：在 workflow 入口增加能力探测与降级提示
4. **静态审计多、运行时遥测少（P2）**
  - 影响类型：持续优化质量
  - 现象：缺少阶段耗时、人工停等、重试轮次等指标
  - 建议：补齐最小遥测并形成周度回看

## 3) 下一轮执行建议（P0/P1/P2）

### P0（立即）

- 建立“效率足够”判定基线：
  - `e2e_total_time`
  - `manual_interventions_per_task`
  - `first_pass_rate`
- 输出要求：每轮必须给出“达标/未达标 + 证据来源”。
- 预期收益：形成统一的效率判定口径，减少评审分歧与重复讨论。
- 主要风险：指标口径定义过粗会误导决策，需先做 1-2 轮校准。

### P1（下一轮）

- 为 `/harness-ship` 引入风险分层执行：
  - 低风险改动：保留 eval/gate，简化高成本覆盖分析
  - 中高风险改动：保持全流程
- 增加入口能力探测：模板声明能力与本地命令能力自动对齐。
- 预期收益：降低低风险改动的交付时长，同时保持核心门禁稳定。
- 主要风险：分层规则设计不当可能造成“误降级”。

### P2（持续）

- 建立 workflow 运行时遥测看板：
  - 阶段耗时分解
  - 人工介入热区
  - 重试/补偿频次
- 用 2-4 周趋势而非单轮数据判断“是否有效提效”。
- 预期收益：把提效从单点经验变成可复盘、可持续优化机制。
- 主要风险：遥测建设初期会增加少量流程与维护成本。

## 4) 风险与边界

- 不以速度牺牲门禁质量；
- 不把“静态评审结论”当作“运行时已优化”的替代；
- 任何提效策略必须保持可回滚与可审计。
